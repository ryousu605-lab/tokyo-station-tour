### B-2. 目視監査ページ `photo-audit.html` の新規作成（公開対象外）

親フォルダ直下に作成する。仕様（この5点のみ実装）:

1. `<script src="scenes.js"></script>` でデータを読み、**地点ごと（scenes の定義順）に4方角の写真をグリッド表示**する。画像パスは先頭の定数 `const DIR = "photos_original/";` を用いる。存在しない画像は `onerror` で「未撮影」表示に差し替える。
2. 各セルのキャプションは `地点名 / 方角ラベル / ファイル名` の3行固定。
3. セルをタップすると赤枠トグル＋ページ下部固定の `<textarea id="marks">` にファイル名を1行ずつ追加/削除する（誤対応の記録用。ユーザーがこのリストをコピーして実装者に渡す）。
4. 各セルに「この写真の実撮影方位: NN°」を表示するため、`tools/exif_report.py`（下記）が生成する `photo-exif.js`（`const EXIF_DIR = {"maru_south_e.jpg": 20.4, ...};`）を `<script>` で読み込む。値が期待方位から±45°を超えるセルには自動で黄色枠を付ける。
5. デザインは素の HTML+CSS でよい（本体のフォント・配色に合わせる必要なし）。fetch は使わない。

`tools/exif_report.py`（そのまま作成）:

```python
from PIL import Image
import os, re, json
DIR = r"..\photos_original"
DECL = -8.0  # 東京の磁気偏角補正
out = {}
base = os.path.join(os.path.dirname(__file__), DIR)
for f in sorted(os.listdir(base)):
    if not f.endswith(".jpg"): continue
    g = Image.open(os.path.join(base, f)).getexif().get_ifd(0x8825)
    if 17 in g: out[f] = round((float(g[17]) + DECL) % 360, 1)
with open(os.path.join(os.path.dirname(__file__), "..", "photo-exif.js"), "w", encoding="utf-8") as fp:
    fp.write("const EXIF_DIR = " + json.dumps(out, ensure_ascii=False, indent=1) + ";\n")
```

（`photo-exif.js` も `.gitignore` / `.vercelignore` に追加する。）

  document.body.classList.add("editing");
  let pinDrag = null;
  const mapInner = document.getElementById("mapInner");
  mapInner.addEventListener("pointerdown", e => {
    const el = e.target.closest(".pin");
    if(!el) return;
    pinDrag = {el, id: el.dataset.id, sx: e.clientX, sy: e.clientY, moved: false};
    el.setPointerCapture(e.pointerId);
  });
  mapInner.addEventListener("pointermove", e => {
    if(!pinDrag) return;
    const r = mapInner.getBoundingClientRect();
    pinDrag.el.style.left = ((e.clientX - r.left) / r.width * 100) + "%";
    pinDrag.el.style.top  = ((e.clientY - r.top)  / r.height * 100) + "%";
    if(Math.abs(e.clientX - pinDrag.sx) > 6 || Math.abs(e.clientY - pinDrag.sy) > 6) pinDrag.moved = true;
  });
  mapInner.addEventListener("pointerup", e => {
    if(!pinDrag) return;
    if(pinDrag.moved && scenes[pinDrag.id]){
      const r = mapInner.getBoundingClientRect();
      const x = editRound((e.clientX - r.left) / r.width * 100);
      const y = editRound((e.clientY - r.top)  / r.height * 100);
      scenes[pinDrag.id].map.x = x;
      scenes[pinDrag.id].map.y = y;
      editCopy(`{x: ${x.toFixed(1)}, y: ${y.toFixed(1)}}`,
        `${scenes[pinDrag.id].name}: {x:${x.toFixed(1)}, y:${y.toFixed(1)}}`);
      const el = pinDrag.el;
      const block = ev => { ev.stopImmediatePropagation(); ev.preventDefault(); el.removeEventListener("click", block, true); };
      el.addEventListener("click", block, true);   // ドラッグ直後の goTo 遷移を1回抑止

    pinDrag = null;

## 5. Phase E — リサイズ＋EXIF全除去（公開用 photos/ の生成）

`tools/build_photos.py` を作成する（そのまま使用）:

```python
from PIL import Image, ImageOps
import pathlib
root = pathlib.Path(__file__).parent.parent
SRC, DST = root / "photos_masked", root / "photos"
DST.mkdir(exist_ok=True)
total = 0
for p in sorted(SRC.glob("*.jpg")):
    im = ImageOps.exif_transpose(Image.open(p))   # 向きを画素に焼き込み
    im.thumbnail((1600, 1600), Image.LANCZOS)     # 長辺1600px（拡大はしない）
    im.convert("RGB").save(DST / p.name, "JPEG",
        quality=82, optimize=True, progressive=True)  # exif=未指定 → 全メタデータ除去
    total += (DST / p.name).stat().st_size
print(f"{len(list(DST.glob('*.jpg')))} files, {total/1e6:.1f} MB")
```

- 目標: 総量 **20MB以下**（1枚平均 150〜300KB 想定）。長辺1600px・quality 82 はスマホ全画面表示で原本と見分けがつかない水準。
- 生成後の検証（必須）: `python tools\verify_photos.py` を作成して実行する。仕様: `photos/*.jpg` 全数について (1) `getexif()` が**空**であること、(2) 長辺 ≤1600px であること、(3) ファイル名が `photos_masked/` と1:1対応であることを assert し、NG 0件で `ALL CLEAR` を出力する。
- `photos/` は git にコミットして公開する（Phase G）。

## 6. Phase F — 移動矢印UIの刷新（視認性＋直観性）

### F-0. 現状の問題（コード解析による）

1. 矢印の回転角が構内図ベクトルそのまま（例: -63.4°）で中途半端に傾き、意図が読みにくい。
2. 階移動（up/down）が「45°斜めの矢印」で、階段だと分からない。
3. 背後・真横のリンクも写真上に放射配置され、「見えている方向へ進む」という直観と合わない。
4. 「戻る」リンクが下向き矢印で表示され、前進の下り方向と区別しにくい。
5. 矢印が半透明の白枠円（60px）で写真に溶けて見づらい。ラベルも等幅10.5pxで小さい。
6. 地点を移動しても視点が常に北向きにリセットされ、「歩いてきた方向」と画面が一致しない。
7. `?edit` のホットスポットドラッグは `links[].x/y` を書き換えるが、通常リンクの表示位置は放射計算で `x/y` を無視するため、**ドラッグしても表示が変わらない**不整合がある。

### F-1. 移動後は「進んだ方向」を向く（問題6）

`goTo()` / `goBack()` の `viewDir = "n";` を次のヘルパー呼び出しに差し替える:

```js
// fromId から toId へ移動したとき向くべき方角（同一フロアの平面移動のみ。階移動・不明時は現状維持）
function faceTravelDir(fromId, toId){
  const a = scenes[fromId] && scenes[fromId].map, b = scenes[toId] && scenes[toId].map;
  if(!a || !b || scenes[fromId].floor !== scenes[toId].floor) return;
  const dx = b.x - a.x, dy = b.y - a.y;
  if(dx === 0 && dy === 0) return;
  const bearing = ((Math.atan2(dx, -dy) * 180 / Math.PI) - MAP_NORTH_DEG + 360) % 360;
  viewDir = ["n", "e", "s", "w"][Math.round(bearing / 90) % 4];
}
```

- `goTo(id)` 内: `viewDir = "n";` を `faceTravelDir(current, id);` に差し替え（`current` 更新より**前**に呼ぶ）。
- `goBack()` 内: `viewDir = "n";` を `faceTravelDir(history[history.length-1], current);` の呼び出しに合わせて差し替える（pop 後の遷移元→遷移先で計算）。

### F-2. 表示の振り分け（問題1・3・4）

- 相対角の正規化ヘルパーを追加:
  ```js
  function relAngle(l){
    let a = linkScreenAngle(l) % 360;
    if(a > 180) a -= 360;
    if(a <= -180) a += 360;
    return a;
  }
  ```
- **写真上に出すのは「前方コーン内（|relAngle| ≤ 90°）の平面リンク」と「階移動リンク」だけ**。矢印の回転角は `Math.round(relAngle(l) / 45) * 45` に**スナップ**する。
- `dir === "back"` のリンクと、|relAngle| > 90° の平面リンクは、画面下部の**行き先バー（#navBar）のチップ**として表示する（back が常に先頭）。
- `renderHotspots()` を次に差し替える:

```js
function renderHotspots(){
  const s = scenes[current];
  const hs = document.getElementById("hotspots");
  hs.innerHTML = "";
  s.links.forEach((l, i) => {
    const stairs = (l.dir === "up" || l.dir === "down");
    if(!stairs && (l.dir === "back" || Math.abs(relAngle(l)) > 90)) return; // 下部バーへ
    const pos = linkScreenPos(l);
    const el = document.createElement("div");
    el.className = "hotspot" + (stairs ? " stairs" : "");
    el.dataset.link = i;
    el.style.left = pos.x + "%";
    el.style.top = pos.y + "%";
    const icon = stairs ? stairsSvg(l.dir) : arrowSvg(Math.round(relAngle(l) / 45) * 45);
    el.innerHTML = `<div class="arrow">${icon}</div><div class="tag">${l.label}</div>`;
    el.onclick = () => goTo(l.to);
    hs.appendChild(el);
  });
  renderNavBar();
}

function renderNavBar(){
  const bar = document.getElementById("navBar");
  bar.innerHTML = "";
  const items = [];
  scenes[current].links.forEach(l => {
    if(l.dir === "up" || l.dir === "down") return;
    if(l.dir === "back"){ items.unshift({l, ic: "↩"}); return; }
    const a = relAngle(l);
    if(Math.abs(a) <= 90) return;
    items.push({l, ic: a > 0 ? "↷" : "↶"});
  });
  items.forEach(({l, ic}) => {
    const c = document.createElement("div");
    c.className = "navChip";
    c.innerHTML = `<span class="ic">${ic}</span>${l.label}`;
    c.onclick = () => goTo(l.to);
    bar.appendChild(c);
  });
  bar.style.display = items.length ? "flex" : "none";
}
```

- `linkScreenPos()` の放射配置行を `return {x: 50 + 32 * Math.sin(rad), y: 56 - 24 * Math.cos(rad)};`（rad は `relAngle(l)` から計算）に変更する（下寄せで「床の上の矢印」感を出す）。
- `arrowRotation()` は不要になるため削除する（呼び出し元も上記で置換済み）。
